## Sections

### 1. Title + one-liner
`# Examageddon` with a one-sentence description: self-hosted exam practice platform.

### 2. Features
Bullet list covering:
- Multiple question types: multiple choice (single/multi-answer), true/false statements, drag-and-drop matching
- Configurable sessions: question order (fixed/random), feedback mode (immediate or at-end review), question count
- Scoring presets including Azure Certification scaled scoring (0–1000)
- JSON bulk import for questions
- Per-person history with pass/fail tracking
- Multi-user (name-based, no accounts)

### 3. Quickstart
Assumes Podman + podman-compose installed.

```bash
git clone <repo>
cd examageddon
podman compose up --build
```

Open `http://localhost:8080`.

Note: data (database + session keys) persists in the `examageddon-db` Docker volume across restarts.

### 4. Configuration
Environment variable table — all can be set in `docker-compose.yml` or passed via `-e`:

| Variable | Default | Description |
|---|---|---|
| `ConnectionStrings__DefaultConnection` | `Data Source=/data/examageddon.db` | SQLite connection string |
| `DataProtection__KeysPath` | `/data/keys` | Directory for ASP.NET session key persistence |
| `ASPNETCORE_ENVIRONMENT` | `Production` | ASP.NET Core environment |

To change the host port, edit the `ports` mapping in `docker-compose.yml` (e.g. `"9090:8080"`).

### 5. Upgrading
```bash
git pull
podman compose up --build
```

The database volume is preserved. `EnsureCreated` is a no-op when the schema already exists.

### 6. Releasing (repo owner)
Tag a commit and publish a GitHub Release. The Actions workflow builds the image and pushes it to `ghcr.io/<owner>/examageddon:<tag>` and `:latest` automatically.
